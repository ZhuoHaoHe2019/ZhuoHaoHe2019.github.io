National Taiwan University

First Week – Learning Problem

From Learning to Machine Learning

learning: acquiring skill from observation

observation ~> learning ~> skill

machine learning: acquiring skill from data

data ~> machine learning ~> skill

But, what is skill ?

skill : improve some performance measure

So we can concrete the definition of ML

data ~> machine learning ~> improved performance measure

Why use machine learning ?

ML: an alternative route to build complicated systems.

​ Just like what we do in our childhood.

Use Scenarios

  • When human cannot program the system manually

    Because the program have to face lots of situation which human cannot preconceive before.

  • When human cannot define the solution easily

    Even people cannot put a list to define the solution.

    such as visual recognition.

  • When needing rapid decisions that humans cannot do

    high-frequency trading

  • When needing to be user-oriented in a massive scale.

    Human cannot know the user-oriented before a user make decisions.

    But machine can analyze much data to predicted.

    such as consumer-targeted marketing.

Key Essence of Machine Learning

help you to decide if or if not you can use ML to solve problem

  1. exists some ‘underlying pattern’ to be learned

    – so ‘performance measure’ can be improved

  2. but no programmable (easy) definition

    – so ‘ML’ is needed

  3. somehow there is data about the pattern

    – so ML has some ‘input’ to learn form

Exam

  • Predicting whether the next cry of the baby happens

    Wrong ! no pattern

  • Determining whether a given graph contains a cycle

    Wrong ! programmable definition easily

  • Deciding whether to approve credit card to some customer

    Yes !

  • Guessing whether the earth will be destroyed by the misuse of nuclear power in the next ten years

    arguably no or not enough data yet.

Applications of Machine Learning

some interesting example of Machine Learning

  1. Food

    • data: Twitter data (word + location)
    • skill: tell food poisoning likeliness of restaurant properly.

  2. Clothing

    • data: sales figures + client surveys
    • skill: give good fashion recommendations to clients

  3. Housing

    • data: characteristics of building and their energy load
    • skill: predict energy load of other building closely

  4. Transportation

    • data: some traffic sign images and meaning
    • skill: recognize traffic signs accurately

  5. Education

    • data: students’ records on quizzes on a Math tutoring system
    • skill: predict whether a student can give a correct answer to anther quiz question.

    Solution

    answer correctly ≈ [recent strength of student > difficulty of question]

    • give ML 9 million records from 3000 students
    • ML determines (reverse-engineers) strength and difficulty automatically

  6. Entertainment: Recommender System

    • data: how many users have rated some movies
    • skills: predicts how a user would rate an unrated movie

    A Possible ML Solution

    • pattern:

      rating <~ viewer/movie factors

    • learning:

      known rating

      ~> learned factors

      ~> unknown rating movies

Components of Machine Learning

Formalize the Learning Problem

input: x ∈ X

output: y ∈ Y

unknown pattern to be learned => target function:

f: X -> Y

data => training examples:

hypothesis => skill with hopefully good performance:

g : X -> Y

  • target f unknown

  • hypothesis g hopefully ≈ f

    but possibly different from f

    The closer between f and g, the better predict result is

Learning model:

Learning Algorithm and Hypothesis Set

Hypothesis Set can contain good or bad hypotheses

up to Learning Algorithm to pick the ‘best’ one as g

A Practical Definition of Machine Learning

Use data to compute hypothesis g that approximates target f

​ S2 = all possible(user id) pairs

​ S3 = all formula that ‘multiplies’ user factors, indexed by all possible combinations of such factors

​ S4 = 1,000,000 pairs of ((user id), rating)

Machine Learning and Other Field

Machine Learning *and *Data Mining

Machine Learning:

use data to compute hypothesis g that approximates target f

Data Mining:

use (huge) data to find property that is interesting

  • if ‘interesting property’ same as ‘hypothesis that approximate target’

    ML = DM

  • if ‘interesting property’ related to ‘hypothesis that approximate target’

    DM can help ML, can vise versa(often, but not always)

  • traditional DM also focuses on efficient computation in large database

Machine Learning and Artificial intelligence

Machine Learning:

use data to compute hypothesis g that approximates target f

Artificial Intelligence:

compute something that shows intelligent behavior

  • g ≈ f is something that show intelligence behavior

    ML can realize AL, among other routes

  • Ml is one possible route to realize AI

Machine Learning and Statistics

Machine Learning:

use data to compute hypothesis g that approximates target f

Statistics:

use data to make inference about an unknown process

  • g is an inference outcome; f is something unknown

    ​ statistics can be used to achieve ML

  • traditional statistics also focus on provable results with math assumptions, and care less about computation

  • many useful statistical tools for ML
